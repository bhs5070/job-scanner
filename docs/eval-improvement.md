# LLM Eval 점수 개선 기록 (0.747 → 0.902)

## 개요

GPT-4o Judge 기반 10개 메트릭 평가에서 평균 점수를 **0.747 → 0.902** (+20.8%)로 개선한 과정을 기록합니다.

## 개선 원인 분석

### 1. Router 프롬프트 보강

**문제**: Router가 사용자 의도를 잘못 분류하면 엉뚱한 에이전트가 답변 → 전체 품질 하락

**변경 내용**:
- Few-shot 예시 5개 → 18개로 확장
- Interview 의도 예시 추가 (기존에 0개)
- 경계 케이스 구분 규칙 6개 명시
  - "풀스택 2년차인데 AI 쪽 이직하고 싶어" → skill_gap (이직 = 역량 갭)
  - "Python 3년 경력으로 어디 지원할 수 있어?" → resume_match (스킬 기반 매칭)
  - "AI Engineer 채용 많아지고 있어?" → trend (시장 동향)

**효과**: 라우터 정확도 향상 → 올바른 에이전트가 처리 → 답변 전체 품질 상승

### 2. 에이전트 프롬프트 튜닝

**문제**: 답변이 두루뭉술하고 다음 행동 제안이 없음 → Judge가 "도움 안 됨"으로 평가

**Search Agent 프롬프트 변경**:
- Before: "간결하게 요약하세요"
- After: 공고별 구조화 (회사+직무+자격요건+특징+URL) + "다음 단계 제안" 섹션 추가

**Match Agent 프롬프트 변경**:
- Before: 공고별 적합도만 나열
- After: 종합 분석 → 공고별 상세 (매칭 강점/부족한 점/지원 전략) → 다음 단계 제안

**Gap Agent 프롬프트 변경**:
- Before: 강점/갭/학습 항목 단순 나열
- After: 즉시(1~2주) / 단기(1~3개월) / 중장기(3개월+) 로드맵 + 구체적 학습 자료 제안

**효과**:
- Helpfulness 0.725 → 0.833 (행동 지향적 답변)
- Answer Completeness 0.750 → 0.944 (구조화로 빠진 부분 없음)
- Relevance 0.750 → 0.944 (질문에 정확히 대응하는 답변)

### 3. 데이터 확장 (334건 → 473건)

**문제**: DB에 공고가 적어서 "신입 ML Engineer" 검색 시 관련 공고가 없음 → 에이전트가 "결과 없음" 반환

**변경 내용**:
- 크롤링 키워드: 3개 → 5개 ("신입 개발자", "주니어" 추가)
- 페이지 수: 1페이지 → 3페이지
- 전체 473건 pgvector 인덱싱 완료

**효과**:
- Retrieval Precision 0.750 → 0.972 (검색 결과가 실제로 관련 있음)
- Retrieval MRR 0.775 → 0.972 (가장 관련 있는 결과가 상위에)
- Context Relevance 0.775 → 0.944 (검색 컨텍스트가 질문에 적합)

## 핵심 인사이트

1. **데이터와 프롬프트는 함께 개선해야 한다**: 데이터가 없으면 프롬프트가 좋아도 검색이 빈약하고, 프롬프트가 나쁘면 데이터가 있어도 답변이 두루뭉술함
2. **Router 정확도가 전체 파이프라인 품질을 결정한다**: 의도 분류가 틀리면 아무리 좋은 에이전트도 엉뚱한 답변을 함
3. **구조화된 답변이 평가 점수를 올린다**: "다음 단계 제안", "로드맵", "종합 분석" 같은 섹션이 Helpfulness/Completeness를 직접 끌어올림
